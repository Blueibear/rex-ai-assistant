# Progress Log

## Learnings
(Patterns discovered during implementation)

- pytest is not installed in the venv; must install with `python -m pip install pytest` first
- The autonomy tests fail due to missing `aiosqlite`; ignore those test files when running broad test passes
- Secrets always come from `os.getenv()` in `build_app_config`; non-secrets come from `_get_nested(json_config, ...)`
- `DEFAULT_CONFIG` in `config_manager.py` and the mapping in `build_app_config` in `config.py` must both be updated for new config sections

## Iteration 6 - US-006: Add TTS voice listing endpoint to Python backend
- Created `rex/tts_voices.py` with `list_voices(provider, *, speakers_dir=None)` covering xtts/edge-tts/pyttsx3
- XTTS: globs `*.wav` from `voices/` dir (project root) or overrideable `speakers_dir`; graceful when dir missing
- edge-tts: calls `edge_tts.list_voices()` via asyncio.run, caches result; `clear_edge_tts_cache()` for tests
- pyttsx3: calls `engine.getProperty('voices')` + `engine.stop()`; graceful ImportError
- All providers return `[]` on ImportError or any exception
- Created `rex_voices_bridge.py` — reads `{"provider": "..."}` from stdin, writes `{"ok": true, "voices": [...]}` to stdout
- Created `tests/test_tts_voices.py` with 13 tests covering all provider paths (mocked)
- Fixed pre-existing regression in US-005: `_build_system_context` used `self._settings` directly, failing for tests that use `Assistant.__new__`; changed to `getattr(self, "_settings", None)`
- Files changed: `rex/tts_voices.py` (new), `rex_voices_bridge.py` (new), `tests/test_tts_voices.py` (new), `rex/assistant.py`
- Learnings for future iterations:
  - Tests that create `Assistant` via `__new__` don't go through `__init__`, so instance attributes set in `__init__` are absent — always use `getattr(self, "_attr", None)` defensively in methods called by such tests
  - edge-tts `list_voices()` is an async coroutine; use `asyncio.run()` to call it from sync context
  - Aliased provider names ("edge_tts", "edgetts") should all resolve to the same path

## Iteration 7 - US-007: Add TTS sample playback endpoint to Python backend
- Added `async def synthesize_sample(provider, voice_id, text)` to `rex/tts_voices.py`
- XTTS: uses `TTS.api.TTS` (behind try/except ImportError), writes to temp WAV, reads back bytes
- edge-tts: uses `edge_tts.Communicate(text, voice_id).save(tmpfile)`, returns MP3 bytes
- pyttsx3: uses `engine.save_to_file + runAndWait`, returns WAV bytes
- All providers enforce 50-char text limit before synthesis
- Created `rex_voice_sample_bridge.py`: reads `{"provider": ..., "voice_id": ...}` from stdin, base64-encodes audio bytes, writes `{"ok": true, "audio_base64": ...}` to stdout
- Added 8 new tests in `tests/test_tts_voices.py` covering edge-tts success, alias spellings, ImportError, text truncation, pyttsx3 success, ImportError, and unknown provider
- Files changed: `rex/tts_voices.py`, `rex_voice_sample_bridge.py` (new), `tests/test_tts_voices.py`
- Learnings for future iterations:
  - For blocking I/O in async functions use `loop.run_in_executor(None, fn)` pattern
  - edge-tts `Communicate.save(path)` is a coroutine; must be `await`-ed inside async function — do NOT wrap with asyncio.run inside it
  - pyttsx3's blocking engine can't be called directly in async context; wrap with executor
