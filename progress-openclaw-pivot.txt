# Progress Log

## Learnings
(Patterns discovered during implementation)

- The retirement check tests have two phases: (1) "NOT SAFE TO RETIRE" tests that track active callers, and (2) after retirement, convert them to regression guards that assert the module stays deleted.
- When retiring a module, also check `tests/test_openclaw_contracts_audit.py` â€” its `CONTRACT_TO_LEGACY` dict maps contracts to legacy modules. Remove the entry when both the contract AND the legacy module are deleted.
- `rex.plugins.load_plugins()` returns `list[PluginSpec]` while the retired `rex.plugin_loader.load_plugins()` returned `dict[str, Any]`. To migrate: build `{f"plugins.{s.name}": s for s in specs}` for backward-compatible dict key access.
- All remaining `[ ]` tasks in the PRD are BLOCKED because OpenClaw is not installed. The only OPENCLAW-REPLACE module that could be retired without OpenClaw was `plugin_loader.py` (since its replacement `rex.plugins` already existed).
