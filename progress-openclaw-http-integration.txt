## Iteration 1 - US-001: Create OpenClaw HTTP client module
- Created `rex/openclaw/errors.py` with OpenClawConnectionError, OpenClawAuthError, OpenClawAPIError all inheriting from AssistantError
- Created `rex/openclaw/http_client.py` with OpenClawClient (post/get/patch/delete, retry logic, auth headers) and get_openclaw_client() singleton
- Created `tests/test_openclaw_http_client.py` with 19 tests using unittest.mock
- Files changed: rex/openclaw/errors.py (new), rex/openclaw/http_client.py (new), tests/test_openclaw_http_client.py (new)
- Learnings for future iterations:
  - The dev venv is at `venv/` not `.venv/` — use `venv/Scripts/pytest.exe` and `venv/Scripts/python.exe`
  - `pydantic` was missing from the venv but installed on demand; may need other deps too
  - mypy complained about `return response.json()` (Any return) — assign to typed variable first: `result: dict[str, Any] = response.json(); return result`
  - ruff UP037 flagged quoted forward refs in `_CLIENT_CACHE: dict[str, "OpenClawClient"]` — fixed automatically with `--fix`
  - ruff I001 flagged import ordering in test file — fixed automatically
  - `requests` is already in pyproject.toml dependencies (>=2.32.0), no new dep needed
  - The `responses` library is NOT in dev deps; use `unittest.mock` for HTTP mocking
